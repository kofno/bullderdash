# Which Documentation File?

## Quick Decision Tree

**I want to get started quickly**
→ `QUICKSTART.md`

**I want to understand what this project does**
→ `README.md`

**I want to use it (all features)**
→ `DOCS.md`

**I want to run commands**
→ `DOCS.md` (Commands section)

**I want to understand the system architecture**
→ `ARCHITECTURE.md`

**I want simulator details**
→ `scripts/sim/README.md`

---

## File Summary

### README.md
- What is bull-der-dash?
- What are its features?
- What was fixed?
- **Read if:** New to the project

### QUICKSTART.md
- How do I get it running?
- What do I need?
- Show me 5 simple steps
- **Read if:** Want to start immediately

### DOCS.md
- How do I use all features?
- What is a queue stat?
- How does the simulator work?
- How do I troubleshoot?
- **Read if:** Need complete guidance

### QUICK_COMMANDS_REFERENCE.md
- What Redis commands exist?
- What APIs are available?
- Show me examples
- What should I see?
- **Read if:** Need command syntax

### ARCHITECTURE.md
- How is the system built?
- What components exist?
- How do they interact?
- Technical deep dive
- **Read if:** Want to understand the code

---

## Pro Tips

✅ **First time?** Start: QUICKSTART.md → DOCS.md

✅ **Need help?** Check DOCS.md (troubleshooting section)

✅ **Stuck on commands?** Use QUICK_COMMANDS_REFERENCE.md

✅ **Want to contribute?** Read ARCHITECTURE.md
